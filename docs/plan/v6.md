# v6 Plan — Draft-First Publication Slice

## Intent
Use v6 to produce the first paper/report draft from the finalized v5.5 canonical baseline.

The canonical evidence base is the complete 10-model `profiled1` set (5 runs/cell). The remaining 4-model backfill stays deferred and non-canonical for this slice.

## Scope decision
- In this repo workflow, v6 is a draft-first milestone.
- PRD security-hardening work (original "Phase 6") is deferred to a later engineering slice/branch.

## Inputs (frozen evidence for v6 draft)
- `results.md` (canonical 10-model tables)
- `docs/reports/v5_5_canonical10_stability.md` (median + IQR companion)
- Repro commands:
  - `python scripts/refresh_profiled1_results.py`
  - `python scripts/check_profiled1_canonical_coverage.py`

## Deliverables

### D1: Draft skeleton and structure (P0)
- Create `docs/paper/draft_v1.md` with stable section layout:
  1. Motivation and benchmark framing
  2. Protocol and setup
  3. Canonical 10-model results
  4. Stability and variance notes
  5. Limitations and deferred scope
  6. Reproducibility appendix pointers

Acceptance:
- Draft has complete section headers and explicit placeholders for all required claims.

### D2: Claim-evidence discipline (P0)
- Create `docs/paper/claims_matrix_v1.md` mapping each key claim to concrete evidence path(s).
- Enforce rule: no claim may depend on deferred 4-model results.

Acceptance:
- Every quantitative statement in `draft_v1.md` is traceable to a row in `claims_matrix_v1.md`.

### D3: Repro appendix assets (P1)
- Create `docs/paper/repro_appendix_v1.md` with exact commands and source-file references.
- Include canonical coverage check output contract (`status=OK` conditions).

Acceptance:
- A new agent can regenerate and verify canonical tables with only documented commands.

### D4: Narrative quality pass (P1)
- Add concise interpretation of model behavior per competition/profile using canonical tables + stability report.
- Document limitations clearly (deferred models, non-security scope).

Acceptance:
- Draft reads as coherent first-pass manuscript, not a raw dump of tables.

## Non-goals (v6)
- Do not resume 4-model backfill as part of draft claims.
- Do not broaden canonical leaderboard scope beyond the 10 complete models.
- Do not start security-hardening implementation in this slice.

## Risks and mitigations
- Risk: accidental inclusion of deferred-model claims.
  - Mitigation: mandatory claims matrix with evidence paths.
- Risk: drift in canonical tables during drafting.
  - Mitigation: run refresh+coverage check before each draft checkpoint.
- Risk: ambiguity between repo v6 and PRD Phase 6.
  - Mitigation: explicitly state draft-first v6 scope in `HANDOFF.md` and this plan.

## Exit criteria for v6
- `docs/paper/draft_v1.md` exists with complete first-pass prose.
- `docs/paper/claims_matrix_v1.md` and `docs/paper/repro_appendix_v1.md` exist and are internally consistent.
- Canonical reproducibility checks pass at final v6 handoff.
- Deferred 4-model status remains documented but excluded from primary claims.

## Execution order
1. Freeze and verify canonical evidence (`refresh` + `coverage check`).
2. Build draft skeleton and claims matrix.
3. Fill results/stability narrative and limitations.
4. Complete repro appendix and perform consistency pass.
