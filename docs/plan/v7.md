# v7 Plan — Post-ArXiv Improvement Slice

## Intent
Use v7 for a focused, moderate-effort upgrade pass on the paper after the first arXiv release.

The goal is to improve practical value and evidence quality without reopening the full benchmark scope.

## Scope
- Add at least one simple non-agentic baseline (AutoML/AutoGluon candidate).
- Integrate new baseline results into the reporting pipeline and paper narrative.
- Make small, high-leverage clarity upgrades to the manuscript where they materially improve interpretation.

## Non-goals
- No broad benchmark redesign.
- No full canonical-policy rewrite.
- No major expansion into new task families for this slice.

## Candidate deliverables

### D1: AutoGluon baseline (P0)
- Add a reproducible AutoGluon baseline path in host-side baseline tooling.
- Support at least the current core paper competitions under existing validation/scoring flow.

Acceptance:
- One command per competition can produce a valid baseline submission via AutoGluon.
- Outputs are compatible with existing validator/scorer and artifact conventions.

### D2: Baseline reporting integration (P0)
- Add baseline rows/summary into the same results surfaces used by the paper.
- Keep provenance explicit (baseline method, version, runtime context).

Acceptance:
- Baseline metrics are reproducible and visible in paper-facing tables/figures or appendix tables.
- Claims matrix links any new quantitative baseline claims to concrete evidence paths.

### D3: Paper revision pass (P1)
- Update paper sections where baseline context improves reader interpretation (results + limitations + reproducibility).
- Keep edits constrained to incremental v1.x improvements.

Acceptance:
- Draft/LaTeX/claims matrix remain consistent.
- PDF builds cleanly and includes any baseline additions.

## Execution order
1. Implement and smoke-test AutoGluon baseline path.
2. Run baseline generation for paper competitions and capture evidence artifacts.
3. Integrate into reporting + claims matrix.
4. Apply minimal paper text/figure/table updates.
5. Rebuild PDF and run final consistency pass.

## Risks and mitigations
- Risk: AutoGluon adds runtime/dependency overhead.
  - Mitigation: keep baseline profile simple and bounded; document exact settings.
- Risk: baseline comparisons are interpreted as definitive.
  - Mitigation: explicitly frame baseline as reference floor/anchor, not exhaustive SOTA tuning.
- Risk: paper drift from evidence.
  - Mitigation: update claims matrix in lockstep with every new quantitative statement.

## Exit criteria
- AutoGluon baseline is reproducible on the benchmark suite used in the paper.
- Baseline evidence is integrated into paper assets and claims mapping.
- Updated paper draft and LaTeX compile successfully with aligned numbers.
